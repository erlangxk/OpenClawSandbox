# First Tool - OpenClaw TypeScript Plugin

This folder is a standalone TypeScript project for an OpenClaw tools plugin.
It adds one tool named weather_current that calls an external API (Open-Meteo).

The plugin now supports plugin-level configuration for API base URLs and lookup language.

## What this project contains

- package.json: independent npm project settings and build scripts
- tsconfig.json: TypeScript compiler settings
- openclaw.plugin.json: plugin manifest and tool contract declaration
- src/index.ts: plugin entry and tool registration
- src/config.ts: plugin config schema and config type
- src/tools/weather-current.ts: weather tool parameters and execution logic

## Plugin configuration

This plugin defines a plugin-level config object. The current supported fields are:

- geocodingBaseUrl: override the geocoding API URL
- forecastBaseUrl: override the forecast API URL
- language: language used for city lookup, defaults to en

Example config shape:

```json
{
   "language": "ja",
   "geocodingBaseUrl": "https://geocoding-api.open-meteo.com/v1/search",
   "forecastBaseUrl": "https://api.open-meteo.com/v1/forecast"
}
```

Use plugin config when:

- you need an API key or endpoint override later
- you want different defaults per environment
- you want tool behavior controlled without changing tool parameters

## Prerequisites

- Node.js 20+
- npm 10+
- OpenClaw CLI available as openclaw command, or access to the OpenClaw gateway container

## Quick start (build and package)

1. Open a terminal in this folder.
2. Install dependencies:
   npm install
3. Type check:
   npm run check
4. Build JavaScript output:
   npm run build
5. Generate plugin tool metadata:
   npm run plugins:build
6. Pack as a local installable tarball:
   npm pack

After step 6 you will get a file like:

- first-tool-0.1.0.tgz

## Install into OpenClaw (host CLI)

Use these if openclaw command is available on your host machine:

1. Install plugin package:
   openclaw plugins install ./first-tool-0.1.0.tgz
2. Enable plugin:
   openclaw plugins enable first-tool
3. Refresh plugin registry:
   openclaw plugins registry --refresh

## Install into OpenClaw (Docker gateway)

Use these if you run OpenClaw in container openclaw-gateway:

1. Copy package into container:
   docker cp ./first-tool-0.1.0.tgz openclaw-gateway:/tmp/first-tool-0.1.0.tgz
2. Install plugin in container:
   docker exec openclaw-gateway node dist/index.js plugins install /tmp/first-tool-0.1.0.tgz
3. Enable plugin:
   docker exec openclaw-gateway node dist/index.js plugins enable first-tool
4. Refresh plugin registry:
   docker exec openclaw-gateway node dist/index.js plugins registry --refresh

## How to test in chat

Ask the agent to call weather_current with a city name, for example:

- weather_current city=Tokyo

Expected result is JSON including city, temperatureC, humidityPct, and windSpeedKmh.

If you set plugin config language to ja, de, fr, or another supported value, the geocoding lookup will use that language.

## How to add more tools

1. Add a new tool block in src/index.ts under tools: (tool) => [ ... ].
2. Add the new tool name to contracts.tools in openclaw.plugin.json.
3. Re-run:
   npm run build
   npm run plugins:build
   npm pack
4. Reinstall the new package version into OpenClaw.

## Important notes

- Do not manually edit openclaw_data/plugins/installs.json. It is generated by OpenClaw.
- Tool names in code must match names in openclaw.plugin.json contracts.tools.
- If OpenClaw reports tool contract errors, run plugins:build again and reinstall.
- Prefer putting shared plugin config in src/config.ts and reading it via the execute function's config argument.
